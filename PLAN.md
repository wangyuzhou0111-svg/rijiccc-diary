# DiaryRecordApp 开发计划

这份计划是一条一条写的，方便以后照着做。这个工具先做成本地网页，因为打开很方便，不需要先学复杂的服务器。

## 第一阶段：先能写

1. 建立共享目录里的 DiaryRecordApp 文件夹。
2. 建立 index.html 页面文件。
3. 建立 styles.css 样式文件。
4. 建立 app.js 脚本文件。
5. 页面顶部放标题、日期、心情、天气。
6. 页面中间放大大的日记编辑区。
7. 页面左边放日记列表。
8. 页面右边放协助工具。
9. 加上保存按钮。
10. 加上新建按钮。

## 第二阶段：让日记不丢

11. 用 localStorage 保存日记。
12. localStorage 是浏览器自己的小仓库。
13. 自动保存草稿。
14. 每次输入后等一小会儿再保存。
15. 显示保存状态。
16. 重新打开页面时，恢复上次写的内容。
17. 给每篇日记一个 id。
18. id 就像每篇日记的身份证号码。
19. 保存创建时间。
20. 保存最后修改时间。

## 第三阶段：让日记更好找

21. 加搜索框。
22. 搜索标题。
23. 搜索正文。
24. 搜索标签。
25. 按日期排序。
26. 显示字数。
27. 显示图片数量。
28. 显示今天是否已经写过。
29. 提供一键跳到今天。
30. 提供清空搜索。

## 第四阶段：让日记更好看

31. 支持调文字颜色。
32. 支持调文字大小。
33. 支持粗体。
34. 支持斜体。
35. 支持下划线。
36. 支持项目符号列表。
37. 支持标题样式。
38. 支持引用样式。
39. 支持一键整理空行。
40. 支持柔和主题色。

## 第五阶段：让日记能放图片

41. 加传图按钮。
42. 选择图片后显示预览。
43. 保存图片的 dataURL。
44. dataURL 是把图片变成一长串文字，方便保存在浏览器里。
45. 可以删除单张图片。
46. 可以给图片写说明。
47. 限制特别大的图片。
48. 图片太大时给中文提示。
49. 图片网格要整齐。
50. 图片不要挡住正文。

## 第六阶段：语音输入

51. 使用浏览器自带的 SpeechRecognition。
52. 如果浏览器不支持，就显示友好提示。
53. 点击麦克风开始听。
54. 再点一次停止。
55. 把听到的话放进正文。
56. 显示正在聆听状态。
57. 语音失败时不清空正文。
58. 语音文字后自动加空格。
59. 让按钮状态清楚。
60. 不偷偷录音，必须用户点击才开始。

## 第七阶段：AI 润色

61. AI 模型目标是 DeepSeek。
62. API 密钥不能写进代码。
63. 先做本地模拟润色。
64. 本地模拟可以修空格、补标点、整理句子。
65. 后面再接真实 API。
66. AI 结果显示在预览框。
67. 用户点接受才替换正文。
68. 用户点取消就保留原文。
69. 提供更通顺模式。
70. 提供保留童真模式。
71. 提供改错别字模式。
72. 提供更有趣模式。

## 第八阶段：写作帮助

## 第九阶段：导出

83. 导出 Markdown。
84. 导出 JSON。
85. Markdown 适合阅读。
86. JSON 适合以后再导入程序。
87. 文件名带日期。
88. 导出时提醒成功。
89. 不自动上传到网络。
90. 保护用户隐私。
